Sources and resources

Everything this project is built on, with links and how we used each one.

Data sources Frontend team: Shubaan, Nia, Joseph, Jeremiah, Dillon

NASA GSFC Eclipse Web Site (Fred Espenak, NASA Goddard)

  • eclipse.gsfc.nasa.gov
  • Lunar eclipse catalog
  • Solar eclipse search

How we used it: every eclipse in our database, including its date, type, magnitude, duration, and Saros number, comes from these NASA Goddard catalogs. This is the verified real source behind all 50 eclipse cards and behind our proof of concept, the 21 August 2017 total solar eclipse.

Our eclipse dataset (the CSV)

  • eclipses_1950_2030_solar_lunar_even_split_fixed_rows.csv converted into content/events.csv

How we used it: 50 eclipses, 25 solar and 25 lunar spread from 1950 to 2030, that we compiled from the NASA GSFC catalogs above, plus the 2017 eclipse and the 2020 great conjunction. Our seeder script loads these into the database that feeds the events panel and the Browse tab.

NASA JPL DE421 planetary ephemeris (de421.bsp)

  • ssd.jpl.nasa.gov

How we used it: this file is the actual table of planetary positions that our code reads. It is produced by NASA JPL and covers 1899 to 2053, which is exactly why our app limits dates to that range. Every dot in the solar system diagram traces back to this file.

NASA JPL Horizons (verification)

  • ssd.jpl.nasa.gov/horizons

How we used it: to spot check that the positions our app calculates match NASA's own published numbers.

Software libraries and tools Backend team: Jonathan, Kiet, Dieuson, Sophia

Skyfield (Python astronomy library)

  • rhodesmill.org/skyfield

How we used it: it reads the DE421 file and does the astronomy. In our position code we ask it where the Earth is looking to see each body, and it hands back the right ascension, declination, and distance. It is the engine behind the diagram.

Flask and flask-cors (Python web framework)

  • flask.palletsprojects.com

How we used it: runs our backend server, serves the web page, and answers the position and event requests.

SQLite (database)

  • sqlite.org

How we used it: stores the events. It is built into Python, so there is nothing to install. Our schema defines the table and our database code queries it by date.

Inspiration

The Antikythera Mechanism

  • en.wikipedia.org/wiki/Antikythera_mechanism

How we used it: the whole concept. It is an ancient Greek device that tracked planetary motion and predicted eclipses, which is what our app does with modern data. That is where the project name comes from.
